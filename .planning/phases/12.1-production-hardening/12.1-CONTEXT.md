<decisions>
## Implementation Decisions

### Authentication (SEC-01)
- **Streamlit native password gate** using `st.secrets` or `.streamlit/secrets.toml`
- Simple shared password — not per-user auth (internal tool for 3-person team)
- Password checked before any page renders
- Fallback: environment variable `APP_PASSWORD` if secrets.toml not present

### SQL Injection Fix (SEC-02)
- **Parameterize SOQL queries** in `providers/salesforce.py`
- Replace string interpolation `f"Website LIKE '%{d}%'"` with escaped values
- SOQL doesn't support bind variables like SQL — use proper escaping (single quotes doubled, special chars stripped)

### CORS/XSRF (SEC-03)
- **Re-enable CORS and XSRF protection** in Dockerfile/docker-compose
- Remove `--server.enableCORS=false --server.enableXsrfProtection=false` flags
- If external access needed, use reverse proxy (Caddy/nginx) instead of disabling protections

### CLI Crash Fix (BUG-01)
- **Fix `create_circuit_breakers()` and `create_rate_limiters()` calls** in `cli/main.py`
- These factory functions take zero arguments but are called with `list(providers.keys())`
- Remove the arguments from both `enrich` and `resume` commands

### Infinite Polling Fix (BUG-02, BUG-03)
- **Add timeout guards** to `providers/icypeas.py` bulk polling loop
- **Add max-iteration guard** to `providers/contactout.py` `_poll_batch_job`
- Default timeout: 300 seconds (5 minutes) for bulk operations
- Default max iterations: 60 polls (with exponential backoff this covers ~10 minutes)

### Settings Persistence (OPS-01)
- **Persist settings changes to .env file** when user modifies them in UI
- Use `python-dotenv` `set_key()` to update .env atomically
- Settings page Save button writes to .env, then `reload_api_keys()` picks up changes
- Waterfall order, budget limits, and provider toggles also persisted

### Docker Hardening (OPS-02)
- **Add non-root USER directive** to Dockerfile
- **Complete .env.example** with all variables: DATAGMA_API_KEY, ANTHROPIC_API_KEY, SALESFORCE_USERNAME, SALESFORCE_PASSWORD, SALESFORCE_SECURITY_TOKEN, APP_PASSWORD

### Claude's Discretion
- Exact Streamlit auth implementation pattern (st.secrets vs custom login page)
- SOQL escaping strategy details
- Whether to add health check endpoint (nice-to-have, not required)
- Exact timeout/backoff values for polling guards

</decisions>
