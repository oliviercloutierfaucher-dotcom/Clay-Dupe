    2. Create ui/pages/emails.py with the full email generation workflow page. Structure:

    **Page header:** "Email Generation" title.

    **Sidebar / top controls:**
    - Campaign selector: `st.selectbox` populated from `db.get_recent_campaigns()`. Show campaign name + status. Only campaigns with status COMPLETED (enrichment done) should be selectable.
    - Template selector: `st.selectbox` populated from `db.get_email_templates()`. Show template name + sequence step. On first load, call `db.seed_default_templates()` to ensure starters exist.
    - Template management expander: `st.expander("Manage Templates")` with:
      - Form to create new template: name, description, system_prompt (text_area), user_prompt_template (text_area), sequence_step (selectbox 1/2/3)
      - List existing templates with delete button
      - Per user decision: `{variable}` syntax explained with available variables list

    **Generation section:**
    - Show count of contacts in selected campaign (from campaign.total_rows)
    - "Generate All" button: launches `threading.Thread(target=run_batch_generation, ...)` with campaign_id, template_id, person_ids from campaign rows, db.db_path, anthropic_api_key. Store thread ref in session_state.
    - "Generate Single" mode: selectbox to pick a specific contact, generates one email inline.
    - Regenerate with note: text_input for user note, regenerate button for selected email.

    **Progress fragment** (reuse enrich.py pattern):
    - `@st.fragment(run_every=2.0)` polling fragment that:
      - Queries `db.get_generated_emails(campaign_id)` to count draft/approved/rejected/failed
      - Shows progress bar: completed / total
      - Shows metric columns: Generated, Failed, Approved
      - Stops polling when all contacts processed (use session_state flag)

    **Email table:**
    - Display generated emails in a table with columns: Contact Name, Company, Subject, Status, Sequence Step
    - Filter by status: All / Draft / Approved / Rejected / Failed (radio buttons or selectbox)
    - Each row expandable (st.expander or click to show details) with:
      - Full email body preview
      - Inline edit: st.text_input for subject, st.text_area for body
      - Save edits button -> calls db.update_email_content()
      - Approve / Reject buttons -> calls db.update_email_status()

    **Bulk actions row:**
    - "Approve All Drafts" button: updates all draft emails to approved
    - "Reject All Drafts" button: updates all draft emails to rejected
    - "Regenerate Selected" (future enhancement placeholder — for now, regenerate one at a time)

    **Export section:**
    - Export preset selectbox: "Outreach.io", "Salesforce Lead", "Raw"
    - Build export DataFrame per preset:
      - **Outreach.io**: Only approved emails. Columns: Email, First Name, Last Name, Company, Subject, Body, Sequence Step. Each sequence step = separate row.
      - **Salesforce Lead**: Only approved emails. Columns: Company, Website, Quality__c, FirstName, LastName, Title, Email, City, State, Country.
      - **Raw**: All emails (including rejected/failed). All available columns.
    - `st.download_button()` with appropriate filename: `emails_outreach_{campaign_name}.csv`, `emails_sf_{campaign_name}.csv`, `emails_raw_{campaign_name}.csv`

    **Key implementation details:**
    - Load ANTHROPIC_API_KEY from settings (os.getenv pattern). If missing, show st.warning with instructions.
    - Database instance from st.session_state.db (set in app.py, same as other pages).
    - asyncio.run() wrapper for all async db calls (established pattern in other pages).
    - For the email table, load Person data to show contact name and email address. Use db.get_person() for each email's person_id, or batch-load persons for the campaign.
    - CAN-SPAM/CASL: Add a st.info note reminding users to add physical address + unsubscribe link in their actual email tool (Outreach handles this).
