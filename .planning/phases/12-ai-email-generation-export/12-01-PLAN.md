    Create data/email_engine.py with:

    1. **Variable substitution** (from 12-RESEARCH.md Pattern 2):
       - `_build_variables(person: Person, company: Optional[Company]) -> dict[str, str]` — maps all template variables from Person + Company models. Handles None fields with empty string fallback.
       - `_score_to_tier(score: Optional[int]) -> str` — Gold >= 80, Silver >= 60, Bronze < 60, Unknown if None.
       - `_substitute_variables(template: str, variables: dict[str, str]) -> str` — regex replacement of {variable} placeholders. Unknown variables left as-is.

    2. **Response parsing:**
       - `_parse_subject_body(response_text: str) -> tuple[str, str]` — parses "Subject: [subject]\n\n[body]" format. Fallback: if no "Subject:" prefix, use first line as subject, rest as body.

    3. **Cost calculation:**
       - `calculate_email_cost(input_tokens: int, output_tokens: int) -> float` — Haiku 4.5 pricing: $1.00/MTok input, $5.00/MTok output.

    4. **Single email generation:**
       - `def generate_single_email(client: anthropic.Anthropic, template: EmailTemplate, person: Person, company: Optional[Company], campaign_id: str, user_note: Optional[str] = None) -> GeneratedEmail` — Builds variables, substitutes in user_prompt_template, calls client.messages.create with claude-haiku-4-5, max_tokens=512, system prompt with cache_control (from Research Pattern 3), parses subject/body, calculates cost, returns GeneratedEmail. On exception: returns GeneratedEmail with status="failed", body=error message.

    5. **Batch generation** (for background thread):
       - `def run_batch_generation(campaign_id: str, template_id: str, person_ids: list[str], db_path: str, api_key: str) -> None` — Creates new Database instance (thread safety), loads template, iterates person_ids, calls generate_single_email for each, saves to DB via asyncio.run(). Adds 1.2s delay between API calls (rate limit safety). This function is the target for `threading.Thread(target=run_batch_generation, ...)`.
